# 统一任务页 — 完整需求文档

**版本**: 1.0  
**更新时间**: 2026-04-02  
**状态**: 设计阶段  
**涉及文件**: `生成详情.html` (改造) + `我的任务.html` (融合)

---

## 一、背景 & 问题分析

### 当前状况
- `生成详情.html` 用聊天气泡展现任务结果 → UI 占空间、信息层级不清
- `我的任务.html` 是表格形式的任务列表 → 交互完整、展示清晰
- 两个页面本质上是同一个数据的两种视图，但菜单、交互、风格都是独立的

### 核心发现
**生成详情页实际上已经变成了任务列表**，因为：
- 每个 Q（用户输入）都是独立的，不带上下文
- 工作流的 Q 包含大量参数信息（不只是几句话）
- 生成结果可能有 100+ 张图片（聊天 UI 展示不下）

### 融合目标
**在一个页面内，通过两种视图模式来展现同一份任务数据，保持内容、交互、操作的统一**

---

## 二、方案概览

### 整体结构

```
┌───────────────────────────────────────────────────────────┐
│ [筛选栏] ← 两态切换（展开模式一行 / 列表模式完整）        │
├───────────────────────────────────────────────────────────┤
│                                                           │
│  [任务 Feed/表格] ← 通过右上角视图切换按钮来回切换       │
│  - 展开模式：默认打开，卡片 feed 风格                      │
│  - 列表模式：紧凑行卡片，信息密集                         │
│                                                           │
├───────────────────────────────────────────────────────────┤
│ [底部输入框] ← 默认收起，可展开                           │
└───────────────────────────────────────────────────────────┘
```

### 页面默认状态
- **默认视图**: 展开模式（用户从通用生成页提交后直接看到任务进度）
- **底部输入框**: 默认收起为单行
- **筛选栏**: 展开模式下收起一行、列表模式下完整展开

## 三、展开模式 (Expanded View)

**默认视图，用于查看单个任务的完整信息。**

### 视觉结构

每个任务是一个**垂直卡片**，INPUT 在上、OUTPUT 在下：

```
┌─ 任务卡片 ─────────────────────────────────────────────────────┐
│                                                               │
│ ┌─ 卡片头 ──────────────────────────────────────────────────┐ │
│ │ [类型Tag] [状态Tag]  创建时间戳  [状态指示]              │ │
│ │ [图像通用生成] [已完成]  2026-03-30 14:32              │ │
│ └─────────────────────────────────────────────────────────┘ │
│                                                               │
│ ┌─ INPUT 区 ────────────────────────────────────────────────┐ │
│ │  "618大促主图，写实风格，白底产品图，要求..."           │ │
│ │  [参考图1][参考图2]             [⊕ 详情] ← hover时显示  │ │
│ │                                                         │ │
│ │  • 通用生成：prompt 文字摘要（2 行截断）                │ │
│ │  • 工作流：工作流名 + 参数概要，hover 显示完整详情      │ │
│ └─────────────────────────────────────────────────────────┘ │
│                                                               │
│ ┌─ OUTPUT 区 ───────────────────────────────────────────────┐ │
│ │  [图1][图2][图3][图4][图5][图6]  共 120 张  [查看全部→] │ │
│ │                                                         │ │
│ │  状态处理：                                              │ │
│ │  • 进行中：显示加载骨架动画 + 脉冲效果                   │ │
│ │  • 已失败：显示失败提示 + 重试按钮                       │ │
│ │  • 已完成：显示结果图网格（前 6-8 张）                  │ │
│ └─────────────────────────────────────────────────────────┘ │
│                                                               │
└───────────────────────────────────────────────────────────────┘
```

### 信息映射表

| 区域 | 内容 | 说明 |
|------|------|------|
| 卡片头 | 类型 Tag（工作流/图像通用/视频通用）、状态 Tag、时间戳、进度指示 | 快速识别任务属性 |
| INPUT | prompt 摘要（2行截断）、参考图缩略图（max 3 + 更多）、详情按钮 | hover 显示"⊕ 详情"查看完整参数 |
| OUTPUT | 结果缩略图网格（6-8张）、总数量、"查看全部"按钮 | 点击"查看全部"跳转详情页 |
| 卡片操作 | 单张图 hover 有下载/入库/再生成按钮 | 快速操作 |

### 状态处理

**进行中** → INPUT 正常显示，OUTPUT 显示骨架屏 + 脉冲加载动画

**已完成未读** → 不显示标记，但任务卡片进入视口时自动标记已读（IntersectionObserver），供后续排序/筛选

**已失败** → OUTPUT 显示失败图标 + 文案 + 重试按钮

## 四、列表模式 (List View)

**紧凑视图，用于快速浏览任务列表、批量筛选、查询历史。**

### 视觉结构

**不是 Ant Design 表格**，而是**紧凑行卡片**，风格与展开模式保持一致：

```
┌─ 任务行 ──────────────────────────────────────────────────────────┐
│ [类型Tag] "任务名/Prompt摘要..."  [输入图x2]→[输出图x3]+117  [状态] │
│ 2h前  [查看详情]                                                  │
└───────────────────────────────────────────────────────────────────┘

┌─ 任务行 ──────────────────────────────────────────────────────────┐
│ [工作流] "大字报裂变工作流"  [输入图x4]→[■■加载中]  [进行中]  5m前 │
└───────────────────────────────────────────────────────────────────┘
```

### 每行从左到右的信息

| 位置 | 内容 | 说明 |
|------|------|------|
| 左 | 类型 Tag（颜色区分） | 快速识别任务类型 |
| 中左 | 任务名 / Prompt 摘要（单行截断） | 任务标题 |
| 中 | 输入缩略图（max 2张）→ 输出缩略图（max 3张）+ 数量 badge | 一目了然 |
| 中右 | 状态 Tag | 已完成 / 进行中 / 已失败 |
| 右 | 相对时间 | 创建/更新时间 |
| 最右 | "查看详情"按钮 | 打开详情弹窗 |

### 行高 & 密度
- **行高**: 56-64px（中档，卡片带圆角和轻微阴影）
- **行间距**: 4-6px（风格与展开模式卡片一致）
- **行卡片样式**: 圆角 + 轻微阴影，不是表格行

### 点击"查看详情"
打开详情弹窗（不切换到展开模式），弹窗内可以：
- 查看完整 Prompt / 工作流参数
- 浏览全部输出图（网格 + 分页 / 懒加载）
- 执行下载 / 入库 / 再生成 操作
- 弹窗关闭后回到列表模式

## 五、筛选栏 (Filter Bar)

### 展开模式 — 收起态 (单行)

```
┌─────────────────────────────────────────────────────────────────┐
│ 🔍 搜索任务...  [全部状态 ▾] [全部类型 ▾] [时间范围 ▾]  [☰│≡] │
└─────────────────────────────────────────────────────────────────┘
```

- **位置**: 页面顶部（不跟随滚动）
- **内容**: 搜索框 + 3 个下拉筛选器（状态、类型、时间）
- **右端**: 视图切换按钮（列表 ☰ / 展开 ≡）

### 列表模式 — 展开态 (卡片)

```
┌──────────────────────────────────────────────────────────────────┐
│ 任务状态 [全部 ▾]   任务类型 [全部 ▾]   创建时间 [📅──📅]      │
│ 创建人 [输入搜索]                                [重置] [☰│≡]    │
└──────────────────────────────────────────────────────────────────┘
```

- **布局**: 多行展开卡片（Ant Design 风格）
- **内容**: 状态、类型、时间范围、创建人、重置按钮
- **右端**: 视图切换按钮

### 筛选器交互规则
- 两种模式的筛选条件**完全一致**，切换视图时**保留筛选状态**
- 搜索框支持模糊查询（任务名、Prompt 摘要）
- 时间范围支持快捷选项（近 7 天 / 近 30 天 / 自定义）

## 六、底部输入框 (Input Box)

### 收起态 (默认)

```
┌─────────────────────────────────────────────────────────────────────┐
│ [+]  描述您想要生成的内容...              [⚡ 生成]             │
└─────────────────────────────────────────────────────────────────────┘
```

- **触发展开**: 点击输入框任意位置
- **样式**: 单行，轻量级

### 展开态

```
┌─────────────────────────────────────────────────────────────────────┐
│ [+上传]  ┌──────────────────────────────────────────────────────┐ │
│          │  textarea: 描述您想要生成的内容...                    │ │
│          └──────────────────────────────────────────────────────┘ │
│                                                                  │
│ [参考图条带]（显示已上传的参考图缩略图）                          │
│ ───────────────────────────────────────────────────────────────── │
│                                                                  │
│ [模型 pill] [比例 pill] [数量 pill] [解析参考图] [⚡ 生成]    │
└─────────────────────────────────────────────────────────────────┘
```

### 交互规则
- **点击外部或按 Esc** → 折回收起态
- **发送后** → 自动折回收起态
- **动画**: 收起 ↔ 展开 平滑过渡（高度动画）
- **上传菜单**: 本地上传、素材库、物料库、原料库、产品物料、我的任务（**目前需要从 6 项简化为 2 项**：本地上传 + 系统资源）

### 弹窗内容

│  └─────────────────────────────────────────────┘  │
│                                                   │

## 八、数据结构

### 任务数据模型

```javascript
{
  id: string,
  name: string,  // 任务名称
  
  type: 'workflow' | 'image_gen' | 'video_gen',  // 任务类型
  status: 'pending' | 'generating' | 'completed' | 'failed',
  
  createdTime: string,  // ISO 8601
  isRead: boolean,      // 已读标记，卡片进入视口时自动置 true
  
  input: {
    // 通用生成
    prompt: string,                // 用户输入的描述文本
    refImages: [{ url, thumb }],   // 参考图列表
    
    // 工作流参数
    workflow: {
      id: string,
      name: string,
      params: {}  // 工作流特定参数
    },
    
    // 生成参数
    model: string,      // 'seedream5' | 'banana2'
    ratio: string,      // '1:1' | '2:3' 等
    res: string,        // '2K' | '4K'
    count: number,      // 生成数量
  },
  
  output: {
    total: number,
    items: [
      { 
        thumb: string,  // 缩略图 URL
        url: string,    // 完整图 URL
        type: 'image' | 'video'
      }
    ]
  }
}
```

## 九、交互规则 & 细节

### 视图切换
- **右上角按钮**: ☰（列表）| ≡（展开）
- **切换时**: 筛选条件保留，滚动位置重置

### 任务排序
- **展开模式**: 按创建时间倒序（最新在上）
- **列表模式**: 同展开模式
- 可考虑"进行中"优先展示

### 响应式
- **移动端**: 列表模式自动调整行信息密度
- **展开模式卡片**: 宽度自适应，最大宽度 860px（参考现有 chat-area）

### 输入框在列表/展开模式下
- **展开模式**: 底部固定，输入框始终可见
- **列表模式**: 底部固定，输入框始终可见（收起态 + 展开态保持一致）

## 十、核心设计原则

1. **统一的内容展示** → 无论哪种视图，任务的关键信息要一致
2. **统一的交互** → 下载/入库/再生成 等操作在两种视图里都可以触发
3. **统一的风格** → 列表行卡片与展开模式卡片风格保持视觉协调
4. **信息层级清晰** → 列表模式快速浏览，展开模式查看细节
5. **性能优先** → 懒加载大图、虚拟滚动处理 100+ 任务
