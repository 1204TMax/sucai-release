### 1.2 生成详情页：状态相关实现未完全对齐

**当前页面现状**

- 状态筛选项未包含 `pending / 排队中`
- 卡片状态映射中存在 `等待中`
- `completed` 在不同位置仍混用“已成功 / 已完成”

**已确认规则**

- 点击“终止”后的统一反馈文案为：已中止

## 2. 已确认但当前页面未必已完整实现

### 2.1 通用生成新增规则

以下规则已经确认，但当前页面未必已经实现到位：

- 解析参考图：
  - 未上传参考图时不可点击
  - 未上传参考图时 hover 提示为：请先上传参考图
  - 上传参考图后 hover 提示为：将参考图解析成提示词
  - 输入框已有提示词时，点击需先弹确认
  - 当前演示逻辑为：确认后进入约 3 秒执行过程，最终提示失败：生成失败，请重试
- 提示词格式化：
  - 无内容时不可点击，但 hover 提示仍显示
  - 执行约 3 秒后，将提示词替换为结构化内容
  - 结构化内容包含：主体、场景、构图、风格、光线、细节、输出要求
- 解析参考图与提示词格式化互斥
- 工具箱一旦点击过内部功能后，鼠标移开不再自动关闭，需点击页面其他区域关闭
- 视频模式使用独立模型集合：
  - Seedance 2.0
  - Seedance 2.0 Fast
  - Vidu Q3 Reference
  - Kling 3.0
  - Kling 3.0 Omni
  - Wan 2.6
- 视频默认模型为：Seedance 2.0
- 视频多模型下，每个模型后的数量单位显示为：个
- 工作流区域应随顶部内容类型切换而同步切换对应类型的工作流入口

### 2.3 全部结果页新增规则

以下规则已经确认，但当前页面还未完整体现：

- `AI 创作/全部图片.html` 与 `AI 创作/全部视频-单模型-加载中.html` 本质上都属于“全部结果页”的不同状态表达
- “全部模型”视图下结果不能混排，需要按模型分组展示
- 每组需要展示该模型下的结果数量
- 即使切到单个模型，该模型标题仍保留
- 只有一个比例时，不显示“全部比例”，只显示唯一比例
- 批量模式下，原有悬停操作入口隐藏
- 批量模式下，每张非加载中卡片右下显示预览按钮，点击只预览、不勾选
- 加载中卡片不显示该预览按钮
- 大图预览内左右切换不能影响底部下载、入库、编辑等操作

## 3. 特殊状态页 / 特殊示例

### 3.1 全部视频-单模型-加载中页

`AI 创作/全部视频-单模型-加载中.html` 属于“全部结果页”的加载中子状态表达。

当前已确认的规则包括：

- 顶部任务信息需体现：视频任务 / 首尾帧 / 5s
- 页面只显示一个模型：Seedance 2.0
- 用户不能切换其他模型
- 结果区同时展示：已成功生成的视频 + 仍在生成中的位置
- 生成顺序按从左到右表现：左侧先出现已成功结果，后续位置仍显示生成中
- 生成过程中不显示“下载全部”与“批量选择”，只显示：中止生成
- 用户点击“中止生成”后，所有尚未成功生成的位置直接消失，只保留已成功结果，并提示：已中止
- 中止后如果只剩一个比例，比例筛选区仍保留，但不显示“全部比例”，只显示唯一比例
- 批量模式规则与普通全部结果页一致：仅非加载中卡片显示预览按钮，加载中卡片不显示
