文档版本：v1.0 · 2026-04-07

编撰人：唐晨越等

## 一、项目概述

### 1.1 项目背景
- AI 创作模块当前已经包含通用生成、任务管理、结果浏览、项目配置、工作流等多个页面入口，但页面之间的职责边界、任务流转关系与结果交付规则尚未以统一文档沉淀。
- 图像生成与视频生成共用一套任务体系，但在输入方式、模型原生支持能力、结果查看方式与下载交付规则上存在差异，若缺少统一定义，容易导致页面行为和研发实现不一致。
- 项目管理、产品物料与提示词优化能力已经实际参与生成链路，但这些能力在什么条件下生效、如何影响结果，目前仍需要统一说明。
- 当前下载结果不是原始产物下载，而是面向投放使用的交付结果，因此尺寸适配、裁剪、压缩、命名、压缩包目录等规则必须提前明确。

### 1.2 项目目标
本项目目标不是单纯补充页面说明，而是明确 AI 创作模块的完整产品能力边界，使业务、产品、设计、研发对同一套规则达成一致。当前版本重点要完成：
1. 明确通用生成、我的任务、全部结果页、项目管理、工作流-裂变 5 个页面的职责划分
2. 明确图像 / 视频任务的统一状态模型与多模型任务聚合规则
3. 明确项目选择、太极提示词优化、尺寸支持判断、结果适配与交付的生成链路
4. 明确单任务结果页的筛选、批量操作、入库与下载规则
5. 明确项目管理页中的品类、产品、产品物料如何作为生成链路的配置源生效

### 1.3 核心设计原则

| 原则 | 说明 |
|---|---|
| 任务统一 | 图像与视频生成共用统一任务体系，统一进入“我的任务”页管理 |
| 页面分工清晰 | 通用生成负责发起，我的任务负责任务管理，全部结果页负责单任务结果浏览，项目管理负责配置源，工作流负责独立发起 |
| 配置前置生效 | 项目选择、模型能力、尺寸支持情况在生成前决定后续处理路径 |
| 结果可交付 | 进入结果页与下载的结果应为可直接投放的素材 |
| 原型辅助理解 | 文档重点写规则、状态、边界与页面分工，具体界面布局与按钮位置以原型为准 |

## 二、系统架构

### 2.1 页面层级结构
系统由 5 个业务页面组成，关系如下：

```text
AI 创作
├── 通用生成              ← 通用图像/视频生成入口
├── 我的任务              ← 我的任务页，统一展示当前用户任务
│    └── 全部图片         ← 单任务结果页（图片/视频统一结果页）
├── 项目管理              ← 品类/产品配置页，提供生成配置数据
└── 工作流-裂变           ← 独立工作流生成入口
```

### 2.2 核心产品对象

| 对象 | 说明 |
|---|---|
| 任务 | 承载一次图像 / 视频生成的完整执行单元 |
| 模型 | 任务下的执行维度，一个任务可关联多个模型 |
| 结果 | 模型产出的图片 / 视频结果，是结果页、下载与入库的基础对象 |
| 项目 | 由品类 / 产品构成，决定项目上下文与提示词优化输入 |
| 产品物料 | 项目管理页维护的 `icon`、`screenshots`、`videos`，可作为生成参考输入 |

### 2.3 关键业务流

```text
项目管理
  └── 维护品类 / 产品 / 产品物料
          ↓
通用生成
  └── 选择项目 / 输入提示词 / 配置模型与尺寸
          ↓
（已选项目时）调用太极优化提示词
          ↓
按模型支持情况生成与适配结果
          ↓
进入我的任务页
          ↓
查看单任务全部结果
          ↓
下载可直接投放的素材
```

### 2.4 页面角色概览

| 页面 | 相对路径 | 页面角色 |
|---|---|---|
| 通用生成 | `AI 创作/通用生成.html` | 通用图像/视频生成入口 |
| 我的任务 | `AI 创作/我的任务.html` | “我的任务”页，统一展示当前用户的通用生成任务与工作流任务 |
| 全部图片 | `AI 创作/全部图片.html` | 单任务结果页，统一展示该任务下全部结果 |
| 项目管理 | `AI 创作/项目管理.html` | 品类/产品配置页，给生成页提供项目与产品物料数据 |
| 工作流-裂变 | `AI 创作/工作流-裂变.html` | 独立工作流生成入口 |

### 2.5 页面跳转关系

| 来源页面 | 操作 | 目标页面 |
|---|---|---|
| 通用生成 | 查看我的任务 | 我的任务 |
| 通用生成 | 进入工作流 | 工作流-裂变 |
| 我的任务 | 查看任务结果 | 全部图片 |
| 我的任务 | 重新编辑通用任务 | 通用生成 |
| 我的任务 | 重新编辑工作流任务 | 工作流-裂变 |
| 工作流-裂变 | 任务生成完成后统一汇入 | 我的任务 |

## 三、统一业务规则

### 3.1 权限模型

#### 3.1.1 功能权限

| 页面 / 能力 | 权限规则 |
|---|---|
| `AI 创作/通用生成.html` | 所有用户可见 |
| `AI 创作/工作流-裂变.html` | 所有用户可见 |
| `AI 创作/项目管理.html` | 仅具备对应功能权限的用户可见 |
| 权限管理页 | 普通用户不可见 |

#### 3.1.2 数据权限

| 层级 | 说明 |
|---|---|
| 品类层 | 控制用户可使用、可管理哪些品类 |
| 产品层 | 控制用户可使用、可管理哪些产品 |
| 账号层 | 某些页面按账号隔离数据 |

补充规则：
- `我的任务.html` 默认是个人任务页，只看自己的任务。
- 该规则不受品类 / 产品权限变化影响：只要是用户自己创建的任务，仍可查看。
- `全部图片.html` 的结果权限跟随任务权限。
- 即使没有产品相关权限，生成能力本身仍可使用；受限的是与具体产品关联的数据能力。

### 3.2 任务类型与实体关系

#### 3.2.1 任务类型

| 任务类型 | 说明 |
|---|---|
| 通用生成 | 固定名称可理解为“通用生成” |
| 工作流 | 使用固定工作流名称，例如“大字报衍生” |

补充规则：
- 通用生成与工作流在更细粒度上都可能落到文本 / 图像 / 视频三种内容类型。
- 本版只覆盖图像、视频。

#### 3.2.2 实体关系

```text
任务
└── 模型（1:N）
    └── 结果（1:N）
```

| 关系 | 说明 |
|---|---|
| 任务 → 模型 | 一个任务可选择多个模型 |
| 模型 → 结果 | 一个模型可生成多条结果 |
| 任务 → 结果 | 一个任务最终对应多条结果 |

### 3.3 任务状态模型

#### 3.3.1 任务状态

| 状态 | 定义 |
|---|---|
| 排队中 | 任务已提交，但尚未开始执行 |
| 生成中 | 任务已开始执行 |
| 已成功 | 任务全部执行完成，且所有模型均成功 |
| 已中断 | 用户手动中止整个任务 |
| 已失败 | 系统异常导致任务失败 |

补充规则：
- 图像任务与视频任务共用同一套任务状态模型。
- 任务提交后自动进入 `AI 创作/我的任务.html`。
- 任务初始可能先进入排队中；如果无需排队，则直接进入生成中。

#### 3.3.2 多模型任务状态聚合规则

多模型任务的整体状态按以下顺序判断：
1. 如果用户手动终止整个任务，整体状态为 **已中断**
2. 否则，只要还有任一模型处于排队或生成中：
   - 若尚未开始执行，则显示 **排队中**
   - 若已有模型开始执行，则显示 **生成中**
3. 否则，只要存在任一失败模型，整体状态为 **已失败**
4. 否则，整体状态为 **已成功**

补充规则：
- 只要还有一个模型仍在生成，任务整体状态就仍为 **生成中**。
- 某个模型失败，不影响其他模型继续生成。
- 已成功产出的结果必须保留。
- 已中断任务中，在中断前已成功产出的结果也必须保留。
- 一旦任务整体状态为 **已成功**，结果不再继续变化。
- 当前版本任务结果按长期保存处理，不设自动清理周期。

### 3.4 下载与交付规则

#### 3.4.1 下载规则
当前版本中，用户下载图片或视频时，客户端下载结果统一为压缩包。

适用范围：
- 单个结果下载
- 批量下载
- 下载全部
- 图片下载
- 视频下载

#### 3.4.2 压缩包命名规则
命名规则：

```text
{姓名标识}_{YYYYMMDD_HHmmss}_{seq}.zip
```

示例：
- `张三_20260407_143025_001.zip`
- `张三1_20260407_143025_001.zip`
- `张三2_20260407_143025_002.zip`

字段说明：

| 字段 | 说明 |
|---|---|
| 姓名标识 | 当前登录用户姓名；若系统内存在重名用户，则在姓名后追加固定区分序号，如 `张三1`、`张三2` |
| `YYYYMMDD_HHmmss` | 发起下载时的系统时间 |
| `seq` | 同一姓名标识在同一秒内多次发起下载时的递增序号，固定为 3 位，从 `001` 开始 |

补充规则：
- 用户姓名允许中文。
- 若用户无重名，则姓名标识直接使用姓名本身。
- 若用户存在重名，系统需为其分配并长期保持固定的重名区分序号。
- 当前阶段不要求在压缩包文件名中体现任务 ID、任务名、模型名、比例等业务信息。

#### 3.4.3 压缩包内目录规则

| 场景 | 规则 |
|---|---|
| 单一模型结果 | 图片/视频文件可直接平铺存放 |
| 多模型结果 | 需按模型维度分文件夹存放 |
| 模型文件夹命名 | 使用模型展示名称；若名称包含不适合作为文件夹名的字符，需做安全转换 |
| 包内文件命名 | 默认保持原始文件名，本期不额外定义重命名规则 |

#### 3.4.4 交付规则
- 进入结果页与下载的结果，应为可直接投放的素材。
- 图片结果在进入结果页与下载前，需压缩至 **400KB 以内**。

### 4.2 链路关键规则

| 节点 | 规则 |
|---|---|
| 是否选择项目 | 选择项目时触发提示词优化链路；未选择项目时不触发 |
| 提示词来源 | 项目模式下使用优化后的提示词；非项目模式下使用用户原始提示词 |
| 尺寸/比例支持 | 若目标尺寸/比例被模型原生支持，则直接生成；否则按系统预设规则处理 |
| 图片结果 | 生成后的图片需压缩至 400KB 以内 |
| 交付结果 | 进入结果页与下载的结果，应为可直接投放的素材 |

## 五、页面详细设计

## 5.1 通用生成页
文件：`通用生成.html`  
入口：AI 创作主入口  
职责：发起图像 / 视频生成任务

### 5.1.1 页面分区

#### 区域一：输入区
用于填写提示词，并补充参考内容。

| 模块 | 说明 | 约束 |
|---|---|---|
| 提示词输入 | 用户输入本次生成提示词 | 必填，为空不可提交 |
| 参考内容 | 支持多来源输入 | 视频模式下存在输入互斥规则 |
| 项目选择 | 选择品类或产品 | 单选 |

参考内容来源：
- 本地上传
- 素材选择弹窗
- 产品物料
- 我的任务

补充规则：
- 各参考输入来源在业务上视为等价来源，进入生成链路后统一处理。
- 视频生成支持普通参考输入，也支持首尾帧模式输入。
- 普通参考输入与首尾帧输入互斥，不可同时存在。

#### 区域二：参数区
用于配置模型、数量、比例、分辨率 / 时长等生成参数。

通用参数规则：

| 项目 | 规则 |
|---|---|
| 单模型数量 | 页面显示统一数量配置 |
| 多模型数量 | 页面显示每模型单独配置数量 |
| 模型下限 | 至少保留 1 个模型 |
| 数量范围 | 1~99 |

图像模式参数：

| 参数 | 说明 | 约束 |
|---|---|---|
| 模型 | 支持多选 | 至少保留 1 个 |
| 数量 / 每模型数量 | 生成数量配置 | 1~99 |
| 比例 | 正式配置项 | 支持常用比例、自定义比例、平台预设尺寸 |
| 分辨率 | 图像清晰度 | 1K / 高清 2K / 超清 4K |
| 项目 | 可选 | 影响提示词优化链路 |

视频模式参数：

| 参数 | 说明 | 约束 |
|---|---|---|
| 模型 | 支持多选 | 至少保留 1 个 |
| 数量 / 每模型数量 | 生成数量配置 | 1~99 |
| 模式 | 视频生成模式 | 智能参考 / 首尾帧 |
| 比例 | 正式配置项 | 支持常用比例和平台预设尺寸 |
| 时长 | 输出时长 | 5秒 / 10秒 / 15秒 / 30秒 |
| 项目 | 可选 | 影响提示词优化链路 |

#### 区域三：辅助能力区

| 能力 | 说明 |
|---|---|
| 我的任务入口 | 可进入 `我的任务.html` |
| 工作流入口 | 可进入 `工作流-裂变.html` |
| 提示词工具箱 | 包含解析参考图、提示词格式化 |

### 5.1.2 项目与提示词优化规则

#### 项目选择规则

| 项目 | 规则 |
|---|---|
| 选择对象 | 可单选一个品类，或该品类下一个产品 |
| 互斥规则 | 不允许同时选择品类和产品 |
| 多选限制 | 不允许多选品类；不允许多选产品 |
| 搜索能力 | 选择器支持搜索品类或产品 |
| 权限裁剪 | 可选范围按用户的品类 / 产品权限裁剪 |
| 无权限展示 | 无权限的品类 / 产品直接不可见 |

#### 太极提示词优化规则
- 当用户选择项目后，提交生成前需调用太极系统统一能力，对提示词进行优化。
- 太极系统内该能力标识为：`genjuxiangmuyouhuatuxiang/shipintishici`
- 该能力适用于图像 / 视频通用生成。
- 若用户未选择项目，则直接使用用户原始提示词。

项目上下文内容定义为：

```text
选中的品类/产品名称：XXXX
用户画像：XXXX
品类描述：XXXX
功能描述：XXXX
核心营销点：XXXX
补充描述：XXXX
```

若任一项为空，则该行不显示（包括标题）。

### 5.1.3 页面状态与提交规则

| 状态 / 场景 | 规则 |
|---|---|
| 提示词为空 | 生成按钮不可用 |
| 提示词非空 | 允许提交 |
| 按钮 loading | 不允许重复提交 |
| 已选择项目 | 提交前先走提示词优化 |
| 未选择项目 | 直接提交原始提示词 |
| 提交后 | 自动进入“我的任务”页 |

### 5.1.4 实现要点
- 内容类型切换后，输入区与参数区需同步切换。
- 图像 / 视频需各自保留独立参数状态，避免切换后丢失输入。
- 项目选择会影响提示词处理链路。
- 图片结果交付前需经过压缩处理。

### 5.1.5 与其他页面关系
- 项目选择数据来源于 `项目管理.html`
- 产品物料选择数据来源于 `项目管理.html`
- 提交后任务自动进入 `我的任务.html`

## 5.2 我的任务页（我的任务页）
文件：`我的任务.html`  
入口：通用生成提交后进入；也可由任务入口进入  
职责：统一展示当前用户的通用生成任务与工作流任务

### 5.2.1 页面分区

#### 区域一：筛选与视图区

| 能力 | 说明 |
|---|---|
| 内容类型切换 | 文本 / 图像 / 视频；本版重点覆盖图像、视频 |
| 搜索提示词 | 按提示词搜索任务 |
| 状态筛选 | 按任务状态过滤 |
| 日期范围筛选 | 支持自定义日期 |
| 模型筛选 | 按模型过滤任务 |
| 项目筛选 | 按项目过滤任务 |
| 展开 / 紧凑视图切换 | 切换任务卡片展示密度 |

#### 区域二：任务卡片区
任务以卡片形式展示，承载以下信息：
- 任务来源
- 任务状态
- 创建时间
- 输入摘要
- 结果摘要
- 提示词摘要
- 参考图摘要（可选）

补充规则：
- 页面上不以独立任务名称作为必展示字段。
- 无匹配任务时显示空状态。
- 左侧提醒区可用于展示进行中任务、已生成未查看。
- 具体卡片视觉结构以原型为准。

#### 区域三：任务操作区

| 操作 | 说明 |
|---|---|
| 重新编辑 | 将原任务参数带回编辑区，但不自动提交 |
| 再次生成 | 沿用原参数直接再次提交 |
| 停止生成 | 中止整个任务，不是只中止某一个模型 |
| 查看全部 | 进入单任务结果页 |

### 5.2.2 状态规则

#### 任务进入规则
- 用户提交任务后，无需手动保存，任务自动进入该页面。
- 页面支持展开、紧凑两种展示密度。

#### 状态流转

| 状态阶段 | 状态 |
|---|---|
| 初始状态 | 排队中 / 生成中 |
| 终态 | 已成功 / 已中断 / 已失败 |

补充规则：
- 已中断仅表示用户手动停止。
- 已失败仅表示系统异常。
- 已成功后任务结果不再继续变化。

#### 各状态允许操作

| 状态 | 允许操作 | 不允许操作 |
|---|---|---|
| 排队中 | 重新编辑、停止生成 | 再次生成 |
| 生成中 | 重新编辑、停止生成 | 再次生成 |
| 已成功 | 重新编辑、再次生成、查看全部 | 停止生成 |
| 已失败 | 重新编辑、再次生成 / 重试、查看全部（有结果时） | 停止生成 |
| 已中断 | 重新编辑、再次生成、查看全部（有结果时） | 停止生成 |

### 5.2.3 页面内输入区
本页底部存在简化版生成入口：
- 默认折叠，可展开 / 收起
- 会随顶部内容类型切换同步切换 placeholder、上传入口和参数区
- 重新编辑 / 再次生成会回填该输入区
- 当前仅体现单模型选择，不体现项目选择能力

### 5.2.4 实现要点
- 页面按个人任务范围查询与展示。
- 多模型任务以任务聚合状态展示，卡片内承载模型级结果摘要。
- 重新编辑与再次生成复用同一套输入回填逻辑。
- 工作流任务与通用生成任务共用统一任务列表结构，通过来源区分。

## 5.3 全部结果页（全部图片）
文件：`全部图片.html`  
入口：我的任务页中查看单任务全部结果  
职责：单任务结果集中浏览、筛选、批量操作

### 5.3.1 页面定位
虽然文件名为“全部图片”，但实际业务含义为统一结果页：

| 任务类型 | 页面展示内容 |
|---|---|
| 图片任务 | 展示该任务下全部图片结果 |
| 视频任务 | 展示该任务下全部视频结果 |

### 5.3.2 功能点

#### 结果范围与查看方式

| 项目 | 说明 |
|---|---|
| 结果范围 | 仅展示当前任务下全部结果，不跨任务聚合 |
| 顶部摘要信息 | 包括项目、来源入口、内容类型、清晰度 / 时长、提示词、参考内容 |
| 模型维度浏览 | 支持按模型查看结果，也支持“全部模型”视图 |
| 默认视图 | 默认展示当前任务全部模型 + 全部比例结果 |

#### 筛选能力
当前筛选维度：
- 模型
- 比例

筛选规则：
- 选择“全部模型”时，结果不能混排，需按模型分组展示。
- 每组显示该模型下的结果数量。
- 即使切换到单个模型，该模型标题仍然保留。
- 模型与比例是组合筛选，不是固定排序规则。
- 结果页中，只有“生成时支持多选的输入项”，才有资格成为筛选项。
- 当前时长不作为筛选项。

#### 批量与结果操作

| 能力 | 说明 |
|---|---|
| 下载全部 | 下载当前任务范围内结果 |
| 批量选择 / 批量下载 | 支持批量结果操作 |
| 批量入库 | 支持批量入库 |
| 单个结果预览 | 支持单个结果预览 |
| 单个结果下载 | 支持单个结果下载 |

补充规则：
- 入库目标包括：素材库、物料库、原料库。
- 当任务为视频任务时，批量选择、下载、入库、删除等操作规则与图片任务一致。
- 图片结果页与视频结果页在页面结构上可理解为同构。
- 浏览态细节以原型为准。

### 5.3.3 页面状态

| 项目 | 规则 |
|---|---|
| 页面状态 | 普通浏览态 / 批量选择态 / 预览态 / 删除确认态 |
| 默认进入状态 | 全部模型 / 全部比例 |
| 默认展示结果 | 当前任务全部结果 |

### 5.3.4 加载中子状态
`AI 创作/全部视频-单模型-加载中.html` 与 `AI 创作/全部图片.html` 本质上都属于“全部结果页”的不同状态表达。

当前规则：
- 单模型视频任务仍在持续生成中时，结果区同时展示已成功结果与生成中位置。
- 生成过程中不显示“下载全部”与“批量选择”，只显示“中止生成”。
- 中止后仅保留已成功结果，并提示“已中止”。

### 5.3.5 实现要点
- 结果页按任务范围查询，不跨任务聚合。
- 全部模型视图下需按模型分组展示结果。
- 图片与视频共用同构结构，仅内容展示形态不同。
- 批量操作、入库与下载均基于当前筛选后的结果范围执行。

职责：维护生成业务所依赖的基础配置数据

### 5.4.1 管理对象
本页只管理两层对象：

```text
品类
└── 产品
```

补充规则：

- 一个产品只属于一个品类。

### 5.4.2 功能点

#### 品类 / 产品树浏览
- 支持浏览、展开与搜索。
- 仅展示当前用户有权限的品类 / 产品范围。
- 无权限产品完全不可见，不以禁用态展示。

#### 品类详情维护
维护以下内容：
- 用户画像
- 品类描述

#### 产品详情维护
维护以下内容：
- 功能描述

维护以下物料：
- `icon`

物料规则：

| 物料类型 | 规则 |
|---|---|
| `icon` | 仅 1 张 |
| `screenshots` | 支持多张 |
| `videos` | 支持多条 |

补充规则：
- 当前只有产品下面有物料，不存在品类级物料挂载。

### 5.4.3 页面状态

| 状态 | 说明 |
|---|---|
| 未选择任何对象 | 右侧显示占位态 |

| 选择产品 | 显示产品详情与产品物料 |

- 项目选择
- 产品物料选择
- 参考图添加链路中的产品物料选择

补充规则：
- 项目选择为单选，可选一个品类，或该品类下一个产品。
- 产品物料在生成链路中统一视为同一种“参考输入”，只是文件类型不同。

### 5.4.5 实现要点
- 项目管理页是生成链路的配置源，不参与任务执行。
- 品类、产品、产品物料需按权限裁剪展示范围。
- 产品物料的选择结果需能被通用生成页直接消费为参考输入。

## 5.5 工作流-裂变页
文件：`工作流-裂变.html`  
入口：通用生成页中的工作流入口  
职责：发起工作流类任务

### 5.5.1 页面定位
工作流-裂变是独立功能，不属于通用生成的子流程，是工作流类任务的发起入口。

### 5.5.2 功能点

#### 工作流输入
当前页面显式提供的输入项包括：
- 参考图上传（支持点击上传与拖拽上传）
- 改图方向（选填）
- 生成数量
- 生图接口

#### 步骤式操作
页面按以下步骤推进：
1. 填写信息
2. 确认提交

#### 提交生成与结果去向
- 提交后生成工作流任务。
- 工作流结果不在本页集中展示，而是统一进入 `我的任务.html`。

补充规则：
- 工作流任务进入统一任务体系后，状态规则、操作规则、结果规则与通用生成任务一致。
- 本版不新增业务逻辑，仅保持原有两步流程与原有交互结构。

### 5.5.3 页面规则

| 项目 | 规则 |
|---|---|
| 参考图要求 | 至少上传 1 张，否则不可进入下一步 |
| 生成数量快捷值 | 3 / 6 / 9 / 12 |
| 自定义数量范围 | 1~50 |
| 生图接口 | 当前为单选卡片 |
| Step 2 摘要 | 展示参考图数量、生成数量、生图接口、改图方向 |
| 改图方向为空 | 确认页可隐藏该行 |

### 5.5.4 实现要点
- 工作流页只负责发起，不承载统一结果浏览。
- 提交成功后需进入统一任务体系，由“我的任务”页继续承接。
- 当前版本应尽量保持原有两步流程结构，不引入额外业务分支。

## 七、当前版本已确认的关键结论
1. 本次 PRD 只覆盖 `AI 创作/` 目录下页面。
2. 我的任务页是“我的任务”页。
3. 全部图片页是单任务结果页。
4. 工作流-裂变是独立功能，但结果统一进入我的任务页。
5. 多模型任务采用任务级聚合状态 + 模型级执行结果的理解方式。
6. 排队中 / 生成中任务不允许再次生成。
7. 项目管理页是配置源页面，支撑通用生成页的数据消费。
8. 图像任务与视频任务共用同一套任务状态模型。
9. 通用生成与工作流都不是用户自定义命名任务。
10. 全部结果页默认展示当前任务的全部模型 + 全部比例结果。
11. 结果页筛选项只来自“生成时支持多选的输入项”。
12. 已失败 / 已中断任务的已成功结果都必须保留。
13. 选择项目时需调用太极能力优化提示词；未选择项目时直接使用原始提示词。
14. 图片结果进入结果页与下载前，需压缩至 400KB 以内。
