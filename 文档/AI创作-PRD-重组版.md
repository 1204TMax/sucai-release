## 三、系统概览
### 3.1 页面结构
AI 创作模块由 5 个页面构成，关系如下：
```text
AI 创作
├── 通用生成              ← 通用图像/视频生成入口
├── 我的任务              ← 统一任务列表页
│    └── 全部图片         ← 单任务结果页（图片/视频统一结果页）
├── 项目管理              ← 品类/产品配置页，提供生成配置数据
└── 工作流-裂变           ← 独立工作流生成入口
```
### 3.2 页面角色
| 页面 | 相对路径 | 页面角色 |
|---|---|---|
| 通用生成 | `AI 创作/通用生成.html` | 通用图像/视频生成入口 |
| 我的任务 | `AI 创作/我的任务.html` | 我的任务页 |
| 全部图片 | `AI 创作/全部图片.html` | 单任务结果页，展示该任务下全部结果 |
| 项目管理 | `AI 创作/项目管理.html` | 品类/产品配置页，给生成页提供配置数据 |
| 工作流-裂变 | `AI 创作/工作流-裂变.html` | 独立工作流生成入口 |
### 3.3 页面跳转关系
| 来源页面 | 操作 | 目标页面 |
|---|---|---|
| 通用生成 | 查看我的任务 | 我的任务 |
| 通用生成 | 进入工作流 | 工作流-裂变 |
| 我的任务 | 查看任务结果 | 全部图片 |
| 我的任务 | 重新编辑通用任务 | 通用生成 |
| 我的任务 | 重新编辑工作流任务 | 工作流-裂变 |
| 工作流-裂变 | 任务生成完成后统一汇入 | 我的任务 |
补充说明：
| 项目 | 说明 |
|---|---|
| 文件名与业务含义 | `全部图片.html` 虽然文件名为“全部图片”，但实际业务含义是统一结果页 |
| 视频任务结果展示 | 当任务为视频任务时，该页同样用于展示该任务下全部视频结果 |
## 四、统一业务规则
### 4.1 权限模型
#### 4.1.1 功能权限
| 页面/能力 | 权限规则 |
|---|---|
| `AI 创作/通用生成.html` | 所有用户可见 |
| `AI 创作/工作流-裂变.html` | 所有用户可见 |
| `AI 创作/项目管理.html` | 仅具备对应功能权限的用户可见 |
| 权限管理页 | 普通用户不可见 |
#### 4.1.2 数据权限
数据权限分三层：
| 层级 | 说明 |
|---|---|
| 品类层 | 控制用户可使用、可管理哪些品类 |
| 产品层 | 控制用户可使用、可管理哪些产品 |
| 账号层 | 某些页面按账号隔离数据 |
补充规则：
| 项目 | 规则 |
|---|---|
| 我的任务页可见范围 | `我的任务.html` 默认是个人任务页，只看自己的任务 |
| 权限变化影响 | 该规则不受品类 / 产品权限变化影响：只要是用户自己创建的任务，仍可查看 |
| 结果页权限 | `全部图片.html` 的结果权限跟随任务权限 |
| 生成能力限制 | 即使没有产品相关权限，生成能力本身仍可使用；受限的是与具体产品关联的数据能力 |
### 4.2 任务类型
任务只有两种大类：
| 任务类型 | 说明 |
|---|---|
| 通用生成 | 固定名称可理解为“通用生成” |
| 工作流 | 使用固定工作流名称，例如“大字报衍生” |
补充规则：
| 项目 | 说明 |
|---|---|
| 内容类型归属 | 通用生成与工作流在更细粒度上都可能落到文本 / 图像 / 视频三种内容类型 |
| 本版覆盖范围 | 只覆盖图像、视频 |
### 4.3 实体关系
任务、模型、结果的数量关系如下：
```text
任务
└── 模型（1:N）
    └── 结果（1:N）
```
即：
| 关系 | 说明 |
|---|---|
| 任务 → 模型 | 一个任务可选择多个模型 |
| 模型 → 结果 | 一个模型可生成多条结果 |
| 任务 → 结果 | 一个任务最终对应多条结果 |
该关系同时适用于通用生成与工作流。
### 4.4 任务状态模型
#### 4.4.1 任务状态
| 状态 | 定义 |
|---|---|
| 排队中 | 任务已提交，但尚未开始执行 |
| 生成中 | 任务已开始执行 |
| 已成功 | 任务全部执行完成，且所有模型均成功 |
| 已中断 | 用户手动中止整个任务 |
| 已失败 | 系统异常导致任务失败 |
补充规则：
| 项目 | 规则 |
|---|---|
| 状态模型复用 | 图像任务与视频任务共用同一套任务状态模型 |
| 提交后去向 | 任务提交后自动进入 `AI 创作/我的任务.html` |
| 初始状态 | 任务初始可能先进入排队中；如果无需排队，则直接进入生成中 |
#### 4.4.2 多模型任务状态聚合规则
多模型任务的整体状态按以下顺序判断：
1. 如果用户手动终止整个任务，整体状态为 **已中断**
2. 否则，只要还有任一模型处于排队或生成中：
   - 若尚未开始执行，则显示 **排队中**
   - 若已有模型开始执行，则显示 **生成中**
3. 否则，只要存在任一失败模型，整体状态为 **已失败**
4. 否则，整体状态为 **已成功**
补充规则：
| 项目 | 规则 |
|---|---|
| 生成中判定 | 只要还有一个模型仍在生成，任务整体状态就仍为 **生成中** |
| 单模型失败影响 | 某个模型失败，不影响其他模型继续生成 |
| 成功结果保留 | 已成功产出的结果必须保留 |
| 中断结果保留 | 已中断任务中，在中断前已成功产出的结果也必须保留 |
| 成功后变化 | 一旦任务整体状态为 **已成功**，结果不再继续变化 |
| 结果保留周期 | 当前版本任务结果按长期保存处理，不设自动清理周期 |
### 4.5 下载与命名规则
当前版本中，用户下载图片或视频时，客户端下载结果统一为压缩包。
#### 4.5.1 适用范围
| 适用场景 |
|---|
| 单个结果下载 |
| 批量下载 |
| 下载全部 |
| 图片下载 |
| 视频下载 |
#### 4.5.2 压缩包命名规则
命名规则：
```text
{姓名标识}_{YYYYMMDD_HHmmss}_{seq}.zip
```
示例：
| 示例文件名 |
|---|
| `张三_20260407_143025_001.zip` |
| `张三1_20260407_143025_001.zip` |
| `张三2_20260407_143025_002.zip` |
字段说明：
| 字段 | 说明 |
|---|---|
| 姓名标识 | 当前登录用户姓名；若系统内存在重名用户，则在姓名后追加固定区分序号，如 `张三1`、`张三2` |
| `YYYYMMDD_HHmmss` | 发起下载时的系统时间 |
| `seq` | 同一姓名标识在同一秒内多次发起下载时的递增序号，固定为 3 位，从 `001` 开始 |
补充规则：
| 项目 | 规则 |
|---|---|
| 用户姓名字符 | 用户姓名允许中文 |
| 无重名处理 | 若用户无重名，则姓名标识直接使用姓名本身 |
| 重名处理 | 若用户存在重名，系统需为其分配并长期保持固定的重名区分序号 |
| 文件名字段范围 | 当前阶段不要求在压缩包文件名中体现任务 ID、任务名、模型名、比例等业务信息 |
#### 4.5.3 压缩包内目录规则
| 场景 | 规则 |
|---|---|
| 单一模型结果 | 图片/视频文件可直接平铺存放 |
| 多模型结果 | 需按模型维度分文件夹存放 |
| 模型文件夹命名 | 使用模型展示名称；若名称包含不适合作为文件夹名的字符，需做安全转换 |
| 包内文件命名 | 默认保持原始文件名，本期不额外定义重命名规则 |
## 五、通用生成链路
### 5.1 链路说明
通用生成（图像 / 视频）遵循统一链路。
#### 5.1.1 流程图
```text
[进入通用生成页]
        |
        v
[用户完成配置]
- 内容类型（图像 / 视频）
- 提示词
- 模型
- 比例/尺寸需求
- 参考内容
- 项目
        |
        v
┌────────────────┐
│ 是否选择项目？ │
└────────────────┘
     | 是                     | 否
     v                        v
[太极优化提示词]            [使用原始提示词]
     \                        /
      \                      /
       v                    v
      [确定最终生成提示词]
                |
                v
[根据目标尺寸/比例 + 模型支持情况，确定处理路径]
                |
                v
┌────────────────────────────┐
│ 目标尺寸/比例是否被模型原生支持？ │
└────────────────────────────┘
     | 是                               | 否
     v                                  v
[按目标规格生成]                    [按可生成规格生成]
                                           |
                                           v
                                      [裁剪/尺寸适配]
                                           |
                                           v
                           ┌────────────────────────┐
                           │ 是否为图片内容？        │
                           └────────────────────────┘
                                | 是                 | 否
                                v                    v
                      [压缩至 400KB 以内]         [按视频规则处理]
                                \                    /
                                 \                  /
                                  v                v
                           [得到可直接投放的素材]
                                        |
                                        v
                               [进入结果页 / 下载]
```
#### 5.1.2 文字说明
```text
进入通用生成页
  → 用户完成配置（内容类型 / 提示词 / 模型 / 比例尺寸需求 / 参考内容 / 项目）
  → 若已选择项目，则调用太极优化提示词；否则直接使用原始提示词
  → 根据目标尺寸/比例 + 模型原生支持情况，确定生成路径
  → 原生支持：按目标规格生成
  → 不原生支持：按可生成规格生成，再做裁剪/尺寸适配
  → 若为图片结果，统一压缩至 400KB 以内
  → 得到可直接投放的素材
  → 进入结果页 / 支持下载
```
### 5.2 链路关键规则
| 节点 | 规则 |
|---|---|
| 是否选择项目 | 选择项目时触发提示词优化链路；未选择项目时不触发 |
| 提示词来源 | 项目模式下使用优化后的提示词；非项目模式下使用用户原始提示词 |
| 尺寸/比例支持 | 若目标尺寸/比例被模型原生支持，则直接生成；否则按系统预设规则处理 |
| 图片结果 | 生成后的图片需压缩至 400KB 以内 |
| 交付结果 | 进入结果页与下载的结果，应为可直接投放的素材 |
以上规则为跨页面统一规则，后续页面章节不再重复展开。

## 七、页面详细设计
## 7.1 通用生成页
**文件：** `AI 创作/通用生成.html`  
**入口：** AI 创作主入口页  
**职责：** 发起标准化图像 / 视频生成任务
### 7.1.1 页面定位
通用生成页用于发起标准化生成任务，本版重点支持图像生成、视频生成。文本生成入口保留在页面结构中，但不纳入本次详细需求。
### 7.1.2 核心功能
#### A. 输入区
本页支持用户输入提示词，并通过参考内容补充生成上下文。
参考内容来源：
| 来源方式 | 说明 |
|---|---|
| 本地上传 | 用户本地文件上传 |
| 素材选择弹窗 | 从系统素材源中选择 |
| 产品物料 | 从项目管理中的产品物料选择 |
| 我的任务 | 从个人既有任务结果中选择 |
关键规则：
| 项目 | 规则 |
|---|---|
| 内容类型 | 页面支持文本、图像、视频切换；本版重点覆盖图像、视频 |
| 来源优先级 | 各参考输入来源在业务上视为等价来源，进入生成链路后统一处理 |
| 视频输入 | 视频生成支持普通参考输入，也支持首尾帧模式输入 |
| 输入互斥 | 普通参考输入与首尾帧输入互斥，不可同时存在 |
#### B. 参数区
图像与视频整体输入结构保持一致，只在必要参数上存在差异。
通用参数规则：
| 项目 | 规则 |
|---|---|
| 单模型数量 | 页面显示独立的“数量”配置 |
| 多模型数量 | 改为每个模型单独配置数量 |
| 多模型自定义 | 每个模型数量可分别自定义 |
| 模型下限 | 至少保留一个模型 |
| 数量范围 | 1~99 |
图像参数：
| 参数 | 说明 |
|---|---|
| 模型 | 支持多选 |
| 数量 | 单模型时配置 |
| 每模型生成数量 | 多模型时配置 |
| 比例 | 正式配置项 |
| 分辨率 | 正式配置项 |
| 项目 | 可选 |
视频参数：
| 参数 | 说明 |
|---|---|
| 模型 | 支持多选 |
| 数量 | 单模型时配置 |
| 每模型生成数量 | 多模型时配置 |
| 模式 | 正式配置项 |
| 比例 | 正式配置项 |
| 时长 | 正式配置项 |
| 项目 | 可选 |
#### C. 辅助能力
| 能力 | 说明 |
|---|---|
| 我的任务入口 | 可进入 `AI 创作/我的任务.html` |
| 工作流入口 | 可进入 `AI 创作/工作流-裂变.html` |
| 提示词工具箱 | 包含解析参考图、提示词格式化 |
#### D. 工作流与工具箱补充规则
| 项目 | 规则 |
|---|---|
| 工作流展开范围 | 本版只展开 `大字报裂变衍生工作流` |
| 工作流联动 | 顶部内容类型切换时，底部工作流入口同步切换 |
| 提示词工具箱 | 解析参考图与提示词格式化不能同时执行 |
| 工具细节 | hover、展开方式等界面行为以原型为准，此处不重复展开 |
### 7.1.3 项目与提示词优化规则
#### 项目选择规则
| 项目 | 规则 |
|---|---|
| 选择对象 | 可以单选一个品类，也可以单选该品类下的一个产品 |
| 互斥规则 | 不允许同时选择品类和产品 |
| 多选限制 | 不允许多选品类；不允许多选产品 |
| 搜索能力 | 选择器支持搜索品类或产品 |
| 取消选择 | 已选项目可取消，回到未选择状态 |
| 权限裁剪 | 可选范围需要按用户的品类 / 产品权限裁剪 |
| 无权限展示 | 无权限的品类 / 产品在项目选择器中直接不可见，不以禁用态展示 |
#### 太极提示词优化规则
项目触发的提示词优化规则已在“五、通用生成链路”中统一定义，此处仅补充变量定义。
变量定义：
- `{{prd}}` 为项目上下文信息，结构如下：
```text
选中的品类/产品名称：XXXX
用户画像：XXXX
品类描述：XXXX
功能描述：XXXX
核心营销点：XXXX
补充描述：XXXX
```
| 变量 | 说明 |
|---|---|
| `{{prd}}` 空值处理 | 若某一项内容为空，则该行整体不显示（包括标题） |
| `{{user_input_prompt}}` | 用户输入的原始提示词 |
### 7.1.4 关键规则与状态
| 维度 | 规则 |
|---|---|
| 提交条件 | 提示词非空时可提交；为空时点击生成将焦点带回提示词输入框；loading 态下不重复提交 |
| 参数状态 | 支持单模型 / 多模型；不同内容类型显示不同参数组合 |
| 项目状态 | 未选择项目 / 已选择项目（单选） |
| 提交后处理 | 会将当前类型、提示词与当前选项快照写入 `pendingGen`，并跳转到 `AI 创作/我的任务.html` |
| 类型切换 | 会按新类型重绘参数区；图像 / 视频 / 文本各自保留自己的参数状态对象 |
### 7.1.5 本页关注信息
| 信息 | 用途 |
|---|---|
| 生成信息 | 承载提示词、模型、项目、参考内容等页面核心内容 |
| 项目信息 | 提供项目选择与项目上下文 |
| 产品物料信息 | 作为参考输入来源之一 |
### 7.1.6 与其他页面关系
| 页面关系 | 说明 |
|---|---|
| 项目选择数据来源 | `AI 创作/项目管理.html` |
| 产品物料选择数据来源 | `AI 创作/项目管理.html` |
| 提交后去向 | 自动进入 `AI 创作/我的任务.html` |

| 操作 | 说明 |

| 能力 | 说明 |

| 项目 | 规则 |

#### 任务进入规则

|---|---|

#### 状态流转
| 状态阶段 | 状态 |
|---|---|
| 初始状态 | 排队中 / 生成中 |
| 终态 | 已成功 / 已中断 / 已失败 |
补充规则：
| 项目 | 规则 |
|---|---|
| 已中断 | 仅表示用户手动停止 |
| 已失败 | 仅表示系统异常 |
| 已成功 | 任务结果不再继续变化 |
#### 各状态允许操作
| 状态 | 允许操作 | 不允许操作 |
|---|---|---|
| 排队中 | 重新编辑、停止生成 | 再次生成 |
| 生成中 | 重新编辑、停止生成 | 再次生成 |
| 已成功 | 重新编辑、再次生成、查看全部 | 停止生成 |
| 已失败 | 重新编辑、再次生成 / 重试、查看全部（有结果时） | 停止生成 |
| 已中断 | 重新编辑、再次生成、查看全部（有结果时） | 停止生成 |
补充规则：
| 项目 | 规则 |
|---|---|
| 结果保留 | 已失败 / 已中断任务中，已成功产出的结果必须保留 |
| 失败态操作区 | 应统一显示卡片底部操作区，不应只在内容区内孤立放一个“重试”按钮 |

| 项目 | 规则 |

| 参数能力 | 为简化版生成入口，能力弱于 `通用生成.html` |
| 模型选择 | 当前仅体现单模型选择 |
| 项目选择 | 当前未体现 `项目` 选择能力 |
### 7.2.5 本页关注信息
| 信息 | 用途 |
|---|---|
| 任务信息 | 承载任务状态、来源、创建时间、所属项目等内容 |

## 九、当前版本已确认的关键结论
1. 本次 PRD 只覆盖 `AI 创作/` 目录下页面。
2. 我的任务页是“我的任务”页。
3. 全部图片页是单任务结果页。
4. 工作流-裂变是独立功能，但结果统一进入我的任务页。
5. 多模型任务采用任务级聚合状态 + 模型级执行结果的理解方式。
6. 排队中 / 生成中任务不允许再次生成。
7. 项目管理页是配置源页面，支撑通用生成页的数据消费。
8. 图像任务与视频任务共用同一套任务状态模型。
9. 通用生成与工作流都不是用户自定义命名任务。
10. 全部结果页默认展示当前任务的全部模型 + 全部比例结果。
11. 结果页筛选项只来自“生成时支持多选的输入项”。
12. 已失败 / 已中断任务的已成功结果都必须保留。
13. 选择项目时需调用太极能力优化提示词；未选择项目时直接使用原始提示词。
14. 图片结果进入结果页与下载前，需压缩至 400KB 以内。
